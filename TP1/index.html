Sélecteurs et Spécificité

⏱ 20 min
1.1 Sélecteurs CSS
Question 1

Sélectionnez tous les liens dans la navigation

✦ Réponse
.main-nav ul a {
  /* Tous les liens <a> dans la nav */
}

On sélectionne les <a> descendants des <ul> dans .main-nav.

Question 2

Sélectionnez uniquement le lien actif

✦ Réponse
.main-nav a.active {
  /* Le <a> avec la classe "active" */
}
Question 3

Sélectionnez l'article qui a la classe featured

✦ Réponse
article.post.featured {
  /* L'article avec les deux classes post ET featured */
}

/* Ou plus court : */
.featured { }
Question 4

Sélectionnez le premier paragraphe de chaque article

✦ Réponse
article.post p:first-of-type {
  /* Premier <p> dans chaque article */
}

/* Ou : */
article.post p:first-child {
  /* Si le <p> est le premier enfant direct */
}
Question 5

Sélectionnez les éléments <li> pairs

✦ Réponse
.main-nav ul li:nth-child(even) {
  /* 2ème, 4ème, 6ème... éléments <li> */
}

/* Équivalent avec 2n : */
.main-nav ul li:nth-child(2n) { }
1.2 Spécificité
Question 1

Classez ces sélecteurs par spécificité croissante : p, .text, #main, p.text, #main .text p

✦ Réponse — Ordre croissant
Rang Sélecteur Score (id, class, tag) Valeur
1 (le plus faible) p (0, 0, 1) 001
2 .text (0, 1, 0) 010
3 p.text (0, 1, 1) 011
4 #main (1, 0, 0) 100
5 (le plus fort) #main .text p (1, 1, 1) 111

Règle de calcul : ID = 100 pts · Classe/pseudo-classe/attribut = 10 pts · Tag/pseudo-élément = 1 pt

Question 2

Si deux règles ont la même spécificité, laquelle gagne ?

✦ Réponse

C'est la règle déclarée en dernier dans le CSS qui gagne (principe de la cascade — ordre d'apparition). La dernière règle écrase les précédentes à spécificité égale.

Question 3

Pourquoi faut-il éviter !important ?

✦ Réponse
  • Il court-circuite la cascade et rend le CSS imprévisible.
  • Il crée une guerre des !important : pour surcharger un !important, il faut en écrire un autre, ce qui devient incontrôlable.
  • Il nuit à la maintenabilité : difficile de comprendre pourquoi un style est appliqué.
  • Alternative : mieux structurer les sélecteurs et la spécificité dès le départ.
02

Box Model

⏱ 25 min
2.1 Comprendre le Box Model
Question 1

Avec box-sizing: content-box (défaut), quelle est la largeur totale visible de .box ?

✦ Réponse
width (contenu)300px
padding gauche + droite (20×2)+ 40px
border gauche + droite (5×2)+ 10px
Largeur totale visible= 350px

Note : le margin (15px) n'est pas inclus dans la largeur visible, il crée l'espace autour.

Question 2

Avec box-sizing: border-box, quelle est la largeur totale visible ?

✦ Réponse
width (inclut padding + border)300px
Largeur totale visible= 300px

Avec border-box, la width englobe le padding et la border. Le contenu interne sera réduit à 300 - 40 - 10 = 250px.

Question 3

Écrivez le reset CSS universel pour box-sizing

✦ Réponse
*,
*::before,
*::after {
  box-sizing: border-box;
}

Le ::before et ::after sont inclus pour couvrir tous les pseudo-éléments générés.

2.2 Centrer un élément
Centrage horizontal + vertical
✦ Réponse
/* ─── Centrage horizontal seulement ─── */
.box {
  width: 600px;
  margin: 0 auto; /* auto gauche+droite */
}

/* ─── Centrage horizontal + vertical (Flexbox) ─── */
body {
  min-height: 100vh;
  display: flex;
  justify-content: center; /* horizontal */
  align-items: center;    /* vertical */
}

/* ─── Alternative : Grid ─── */
body {
  min-height: 100vh;
  display: grid;
  place-items: center;
}

CSS Grid

⏱ 30 min
4.1 Grille de cartes responsive
Code CSS + Résultat
✦ Réponse
.grid-cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 16px;
}

/*
  auto-fit  → crée autant de colonnes que possible
  minmax    → chaque colonne fait min 200px, max 1fr
  Résultat  → s'adapte automatiquement à la fenêtre
*/

.grid-cards .card {
  background: #1e293b;
  border-radius: 8px;
  padding: 24px;
  text-align: center;
}

Responsive Design — Mobile First

⏱ 25 min
Media Queries Mobile First
✦ Réponse complète
/* ═══ BASE : Mobile (0px → 767px) ═══ */
.container {
  width: 100%;
  padding: 15px;
}

.nav-menu {
  display: flex;
  flex-direction: column; /* liens empilés sur mobile */
  gap: 5px;
}

.content {
  display: block; /* une seule colonne */
}

.sidebar {
  display: none; /* masquée sur mobile */
}

/* ═══ TABLETTE : min-width 768px ═══ */
@media (min-width: 768px) {

  .container {
    max-width: 768px;
    margin: 0 auto;
    padding: 20px;
  }

  .nav-menu {
    flex-direction: row; /* liens horizontaux */
    gap: 16px;
  }

  .content {
    display: grid;
    grid-template-columns: 1fr 1fr; /* 2 colonnes sur tablette */
    gap: 20px;
  }

  .sidebar {
    display: block; /* réapparaît */
  }
}

/* ═══ DESKTOP : min-width 1024px ═══ */
@media (min-width: 1024px) {

  .container {
    max-width: 1200px;
    padding: 40px;
  }

  .content {
    grid-template-columns: 240px 1fr; /* sidebar + main */
  }
}

L'approche Mobile First utilise toujours min-width pour les breakpoints : on part des styles de base (mobile), puis on enrichit pour les écrans plus grands.

Système de couleurs clair / sombre
✦ Réponse
/* ═══ Thème clair (défaut) ═══ */
:root {
  --color-bg:        #f8fafc;
  --color-surface:   #ffffff;
  --color-text:      #0f172a;
  --color-text-muted:#64748b;
  --color-primary:   #3b82f6;
  --color-border:    #e2e8f0;
  --radius:          8px;
  --shadow:          0 2px 8px rgba(0,0,0,.08);
}

/* ═══ Thème sombre ═══ */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg:        #0f172a;
    --color-surface:   #1e293b;
    --color-text:      #e2e8f0;
    --color-text-muted:#94a3b8;
    --color-primary:   #60a5fa;
    --color-border:    #334155;
    --shadow:          0 2px 8px rgba(0,0,0,.3);
  }
}

/* ═══ Bascule manuelle via data-theme ═══ */
[data-theme="dark"] {
  --color-bg:      #0f172a;
  --color-surface: #1e293b;
  /* … même valeurs … */
}

/* ═══ Utilisation des variables ═══ */
body {
  background: var(--color-bg);
  color:      var(--color-text);
}

.card {
  background:    var(--color-surface);
  border:        1px solid var(--color-border);
  border-radius: var(--radius);
  box-shadow:    var(--shadow);
}

.btn-primary {
  background: var(--color-primary);
  color:      #fff;
}

Transitions et Animations

⏱ 20 min
7.1 Bouton animé
Code CSS + Résultat interactif
✦ Réponse
.btn {
  padding:         14px 32px;
  border:          2px solid #6ee7b7;
  background:      transparent;
  color:           #6ee7b7;
  border-radius:   8px;
  cursor:          pointer;
  font-weight:     700;

  /* Transition sur plusieurs propriétés */
  transition:
    background .3s ease,
    color .3s ease,
    transform .2s ease,
    box-shadow .3s ease;
}

.btn:hover {
  background:  #6ee7b7;
  color:       #000;
  transform:   translateY(-3px);
  box-shadow:  0 12px 30px rgba(110,231,183,.35);
}

.btn:active {
  transform:   translateY(0);
  box-shadow:  none;
}
7.2 Animation de chargement (Spinner)
Code CSS + Résultat animé
✦ Réponse
/* ─── Spinner classique ─── */
@keyframes spin {
  to { transform: rotate(360deg); }
}

.spinner {
  width:        48px;
  height:       48px;
  border:       3px solid rgba(110,231,183,.2);
  border-top:   3px solid #6ee7b7;
  border-radius:50%;
  animation:   spin 0.8s linear infinite;
}

/* ─── Pulse ring ─── */
@keyframes pulse-ring {
  0%   { transform: scale(.8);  opacity: 1; }
  100% { transform: scale(1.4); opacity: 0; }
}

/* ─── Dots bounce ─── */
@keyframes dots-bounce {
  0%, 80%, 100% { transform: scale(0); }
  40%           { transform: scale(1); }
}
